
# Engineering Decisions

## Why Streamlit?

I chose Streamlit because I wanted to quickly turn an idea into a working prototype.

The priority was testing:
"Can this actually help students?"

A more complex frontend would come later.

---

## Why SQLite?

SQLite was chosen because it is lightweight and simple.

For a prototype, it allowed me to:
- store conversations
- test features
- iterate quickly

A production version would move to a larger database.

---

## Why use an existing AI model?

Training a large AI model from scratch is not realistic for a student project.

Instead, I focused on the engineering around AI:

- prompting
- user experience
- learning workflows
- personalisation
- system design

---

## Lessons Learned

The biggest lesson from StudyForge:

Building an AI product is not only about the model.

The difficult parts are:
- understanding users
- designing systems
- handling data
- creating useful experiences
